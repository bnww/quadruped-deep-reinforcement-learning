# Quadruped Robot Locomotion via Reinforcement Learning

A PyTorch implementation of the **TD3 (Twin Delayed Deep Deterministic Policy Gradient)** algorithm applied to quadruped robot locomotion control. Trained on two environments of increasing complexity: the MuJoCo **Ant** (8 degrees of freedom) and ANYbotics' **ANYmal** (12 degrees of freedom) in the Gym Jiminy physics simulation.

The goal: train quadruped robots to stand and walk forward from scratch — with no hand-coded movement rules, only a reward signal and hundreds of thousands of simulated episodes.


## Results

### Ant (8-DOF)

<img src="training_video/Ant_image.jpg" width="300" alt="Ant Robot">

The TD3 agent learned rapidly in the Ant environment, achieving consistently positive rewards within the first 100 episodes. After 5,000 episodes of training, the agent achieved a **maximum reward of ~4,800** and a **mean reward of 1,309.8** — compared to a mean of -57.7 for randomly sampled actions. The trained agent moves quickly and purposefully to the edge of its environment.

### ANYmal (12-DOF)

<img src="training_video/Anymal_image.jpg" width="300" alt="ANYmal Robot">

ANYmal was a significantly harder problem. Unlike Ant, the ANYmal robot collapses immediately under random actions (average survival: 0.45 seconds), making the initial learning phase extremely challenging — the agent had to learn to stand before it could learn to walk.

After **200,000 episodes** of training, the TD3 agent successfully learned to keep the ANYmal robot standing for up to **20 seconds** (the maximum simulation duration), with tentative forward locomotion. The core challenge was balancing competing reward signals: maximising survival penalised movement, and vice versa.

| Metric | TD3 Agent | Random Actions |
|--------|-----------|----------------|
| Mean total reward | 131.5 | -57,886.0 |
| Mean survival time (s) | 19.5 | 0.45 |
| Mean travel distance | 0.31 | 0.00033 |

## Why TD3?

Both environments have **continuous action spaces** — at every timestep, the agent outputs precise torque values for each joint simultaneously. This rules out discrete-action algorithms like DQN.

TD3 was chosen over the simpler DDPG baseline for three key reasons:

- **Twin critics** reduce Q-value overestimation, a common failure mode in actor-critic methods that causes policies to exploit inaccurate value estimates and collapse
- **Delayed policy updates** prevent the actor from overfitting to noisy critic estimates early in training — the policy is updated only every 3 critic updates
- **Target policy smoothing** adds clipped noise to target actions, acting as a regulariser that improves generalisation across nearby states

We also implemented **Soft Actor-Critic (SAC)** but were unable to achieve improved results over TD3 on these environments, and experimented with **prioritised experience replay** as an alternative to uniform sampling from the replay buffer.

## Environments

### Ant (MuJoCo Gym)

A simple 8-DOF quadruped. Relatively forgiving — the robot survives even under random actions, making initial learning straightforward.

- **State space**: 27 continuous variables — torso position, orientation, linear and angular velocity, plus joint positions and velocities
- **Action space**: 8 continuous torque values in (-1, 1), one per joint
- **Reward function** (predefined): survival bonus (torso Z height within [0.2, 1.0]) + forward movement in X direction − action magnitude cost − contact force cost

### ANYmal (Gym Jiminy)

ANYbotics' ANYmal robot — a 12-DOF quadruped with realistic rigid body dynamics. Significantly more complex: immediate collapse under random actions, and no predefined reward function provided by the environment.

- **State space**: 37 continuous variables — 3D position and orientation of the torso (7), joint positions for 12 actuated joints (hip abductors, hip flexors, knee flexors × 4 legs), plus linear/angular velocities for all
- **Action space**: 12 continuous torque values in (-80, 80), one per actuated joint
- **Reward function** (custom-designed): survival bonus (+1 per timestep) + forward movement reward + smooth action reward − death penalty (-10 on collapse). Reward shaping was a key challenge — balancing survival against forward movement required significant experimentation

## Architecture

Both environments use the same TD3 structure: twin critic networks, actor and target actor, with Huber loss for critic training and Polyak averaging for target network updates.

### Ant

| Component | Architecture |
|-----------|-------------|
| **Actor** | Input(27) → Dense(256, ReLU) → Dense(128, ReLU) → Output(8) |
| **Critics (×2)** | Input(35) → Dense(256, ReLU) → Dense(128, ReLU) → Output(8) |

| Hyperparameter | Value |
|----------------|-------|
| Episodes | 5,000 |
| Batch size | 128 |
| Replay buffer size | 5,000 |
| Policy learning starts | Episode 1,000 |
| Policy update delay | Every 3 critic updates |
| Actor learning rate | 1e-4 |
| Critic learning rate | 1e-4 |
| Discount factor (γ) | 0.95 |
| Polyak update rate (τ) | 0.05 |

### ANYmal

| Component | Architecture |
|-----------|-------------|
| **Actor** | Input(37) → Dense(256, ReLU) → Dense(128, ReLU) → Output(12) |
| **Critics (×2)** | Input(49) → Dense(256, ReLU) → Dense(128, ReLU) → Output(12) |

| Hyperparameter | Value |
|----------------|-------|
| Episodes | 200,000 |
| Batch size | 64 |
| Replay buffer size | 5,000 |
| Policy learning starts | Episode 1,000 |
| Policy update delay | Every 3 critic updates |
| Actor learning rate | 1e-5 |
| Critic learning rate | 1e-5 |
| Discount factor (γ) | 0.95 |
| Polyak update rate (τ) | 0.05 |

Key differences for ANYmal: **10× lower learning rate** (1e-5 vs 1e-4) and **smaller batch size** (64 vs 128) were needed to combat instability when learning to control a more complex robot with a harder reward landscape. The much larger episode count (200,000 vs 5,000) reflects the difficulty of the problem — the agent spent the first ~140,000 episodes simply learning to survive before meaningful forward locomotion emerged.

## Key Technical Decisions

**Huber loss over MSE**: We used Huber loss for critic training to improve stability. Huber loss is less sensitive to outliers than MSE, which helps when the TD-targets are noisy early in training.

**Custom reward shaping for ANYmal**: The Gym Jiminy environment provides no predefined reward function for ANYmal. Designing a reward that balanced survival, forward movement, and smooth control was one of the hardest parts of the project. The -10 death penalty was critical — without it, the agent had insufficient incentive to learn to stand before attempting to move.

**Conservative hyperparameters for ANYmal**: The 10× lower learning rate for ANYmal wasn't arbitrary — at higher learning rates, the policy would periodically collapse catastrophically after making progress, losing learned standing behaviour. The conservative rate produced slower but more stable convergence.
